# Contributing to Open Human Preference Modeling

We welcome contributions to the Open Human Preference Modeling project!

## Getting Started

1.  Fork the repository.
2.  Clone your fork: `git clone https://github.com/your-username/OpenHumanPreferenceModeling.git`
3.  Install dependencies (see [README.md](README.md)).

## Development Workflow

1.  Create a new branch for your feature or fix: `git checkout -b feature/my-new-feature`
2.  Make your changes.
3.  Run tests to ensure no regressions:
    -   Backend: `pytest`
    -   Frontend: `npm test`
4.  Commit your changes with clear messages.
5.  Push to your fork and submit a Pull Request.

## Code Style

-   **Python**: Follow PEP 8. We use `black` and `isort`.
-   **TypeScript**: Follow the existing style. We use `eslint` and `prettier`.

## Pull Request Process

1.  Ensure your PR description clearly describes the problem and solution.
2.  Link to any relevant issues.
3.  Wait for review from maintainers.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
